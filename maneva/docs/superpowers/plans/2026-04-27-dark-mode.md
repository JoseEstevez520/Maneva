# Dark Mode — Plan de Implementación

**Goal:** Implementar dark mode profesional con tokens semánticos. El usuario activa/desactiva manualmente desde Ajustes. El tema persiste entre reinicios.

**Architecture:** Tokens semánticos en `tailwind.config.js` (nombres de rol, no de color). `DarkColors` paralelo en `theme.ts`. Hook `useThemeColors()` para props JS. Puente `uiStore → NativeWind` en `_layout.tsx`.

**Tech Stack:** NativeWind 4.2.3, Zustand persist, safeStorage, Expo Router

## Tarea 3 — `src/hooks/useThemeColors.ts`: nuevo hook

**Fichero:** `src/hooks/useThemeColors.ts` (crear)

- [ ] Crear el fichero:

```ts
import { useUiStore } from '@/store/uiStore'
import { Colors, DarkColors } from '@/constants/theme'

/**
 * Devuelve el objeto de colores correcto según el tema activo.
 * Usar en componentes que necesitan colores en props JS (iconos, estilos inline).
 * Para clases Tailwind usar dark: prefix directamente.
 */
export function useThemeColors() {
  const colorScheme = useUiStore(s => s.colorScheme)
  return colorScheme === 'dark' ? DarkColors : Colors
}
```

import { create } from 'zustand'
import { persist, createJSONStorage } from 'zustand/middleware'
import { safeStorage } from '@/lib/storage'

type ColorScheme = 'light' | 'dark'

type UiStore = {
  colorScheme: ColorScheme
  hasSeenOnboarding: boolean
  setColorScheme: (scheme: ColorScheme) => void
  setHasSeenOnboarding: (seen: boolean) => void
}

export const useUiStore = create<UiStore>()(
  persist(
    (set) => ({
      colorScheme: 'light',
      hasSeenOnboarding: false,
      setColorScheme: (colorScheme) => set({ colorScheme }),
      setHasSeenOnboarding: (hasSeenOnboarding) => set({ hasSeenOnboarding }),
    }),
    {
      name: 'ui-store',
      storage: createJSONStorage(() => safeStorage),
    }
  )
)
